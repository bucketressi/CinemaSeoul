## 브랜치 사용

* 각자의 github repo에서는 자유롭게 브랜치 사용
* PR 넣을 때는 frontend, backend 나누어서 넣고
  * backend는 1명이므로 별도 코드리뷰 없이 본인이 merge
  * frontend는 2명이므로 상대방이 코드리뷰 후 merge
* 각 기능이 완료되었다고 느껴질때는 main에 PR 후 전부 코드리뷰 후 merge
